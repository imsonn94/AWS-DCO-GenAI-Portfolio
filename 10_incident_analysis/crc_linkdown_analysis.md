# CRC Error·Link Down 교육용 Incident 분석

- 작성자: 손봉교

- 사용 파일: sample_dco_log.txt
- 사용 자료: 교육용 샘플 데이터

## 1. 분석 대상

- 날짜: 2026-07-03
- 샘플 장비명: SAMPLE_TOR_SW_01
- 인터페이스: Gi0/1
- Ticket ID: EDU-TKT-2026-0003
- 분석 시간: 03:05~03:30

## 2. 사건 전 상태

- 사건 발생 전(03:02) 로그에서는 SAMPLE_TOR_SW_01은 정상적으로 Normal heartbeat를 주고받고 있었으며, 별도의 경고나 이상 징후는 없었음

## 3. 시간순 사건

| 시간 | 이벤트 | 확인한 내용 |
|---|---|---|
| 03:05 | CRC error 증가 | Gi0/1 인터페이스의 CRC 에러 카운터가 5분 만에 154회로 증가 |
| 03:06 | Link Down | Gi0/1 인터페이스 상태가 DOWN으로 변경되고 서버 연결이 끊김 |

| 03:25 | Maintenance completed | LC-LC 멀티모드 패치 코드 교체 및 포트 커넥터 청소 완료 |

## 4. 로그에서 확인된 사실

- SAMPLE_TOR_SW_01 장비의 Gi0/1 인터페이스에서 CRC 에러 카운터가 짧은 시간(5분) 내 154회로 증가함
- CRC 에러 증가 직후(1분 이내) 같은 인터페이스가 DOWN 상태로 전환됨
- 티켓이 개설된 뒤 현장 케이블링팀으로 이관됨
- 패치 코드 교체와 커넥터 청소 작업이 이루어짐
- 작업 완료 후 인터페이스가 UP 상태로 전환되었고, CRC 에러가 0건으로 확인됨

## 5. 가능한 원인

### 가능성 1

- 가능한 원인: 패치 케이블 또는 커넥터의 접촉 불량
- 로그에서 찾은 근거: 조치 내용이 "패치 코드 교체"와 "커넥터 청소"였고, 조치 이후 문제가 해결됨
- 추가로 필요한 정보: 교체 전 커넥터의 실제 상태(육안으로 접촉불량, 오염 등 확인 결과)

### 가능성 2

- 가능한 원인: 케이블 자체의 물리적 손상(꺾임, 노후화 등)
- 로그에서 찾은 근거: CRC 에러(데이터 손상 검출) 증가가 Link Down보다 먼저 관찰됨
- 추가로 필요한 정보: 교체된 케이블의 손상 여부에 대한 점검 기록

### 가능성 3

- 가능한 원인: 인터페이스(포트) 자체의 하드웨어 이상
- 로그에서 찾은 근거: 로그만으로는 케이블 문제인지 포트 자체 문제인지 완전히 구분되지 않음
- 추가로 필요한 정보: 포트 교체 없이 케이블을 교체하는 조치만으로 문제가 해결되었는지에 대한 확인 필요

## 6. 현재 로그만으로 확인할 수 없는 내용

- 케이블이 실제로 손상되어 있었는지 여부
- 커넥터 오염/먼지 등 구체적인 원인
- 포트 자체의 하드웨어 결함 여부
- 이번이 처음 발생한 문제인지, 과거에도 반복된 적이 있는지

## 7. 추가로 확인하면 좋은 정보

1. 같은 인터페이스(Gi0/1)에서 과거에도 CRC 에러나 Link Down 이력이 있었는지
2. 교체 전 케이블의 실제 상태(육안 점검 결과 등)
3. 같은 시간대 인근 다른 장비에서도 유사한 이벤트가 있었는지
4. 작업 담당자가 남긴 추가 소견(있다면)

## 8. 사건 전·중·후 요약

- 사건 전: 정상적으로 Normal heartbeat 신호를 주고받던 상태
- 사건 중: CRC 에러 증가 → 인터페이스 다운 → 티켓 개설·이관 → 케이블 교체 및 청소
- 사건 후: 인터페이스 정상 복구, CRC 에러 0건으로 확인됨

## 9. 한 문장 Incident 요약

- SAMPLE_TOR_SW_01 장비의 Gi0/1 인터페이스에서 CRC 에러 발생 증가 후 연결이 끊겼고, 패치 케이블 교체 및 커넥터 청소 작업 후 정상 복구되었으며, 추후 정확한 원인파악이 필요함.

## 10. 사용한 AI 프롬프트

- 역할: 공개 직무 맥락을 기반으로 교육용 DCO 로그를 분석하는 학습 도우미
- 목표: 로그에서 확인된 사실, 가능한 원인, 미확인 정보, 추가로 확인하면 좋은 정보를 구분해서 정리
- 조건: 교육용 샘플 데이터임을 표시, 원인 하나로 확정하지 않기, 로그에 없는 내용을 사실처럼 쓰지 않기, 실제 장비 조작 절차 작성 금지, 비전공자도 이해할 수 있는 짧은 문장으로 작성
