## 3. 교육용 사건 개요

2026년 7월 3일 새벽, 샘플 스위치 장비(`SAMPLE_TOR_SW_01`)의 특정 포트(`Gi0/1`)에서  
데이터 오류 횟수가 급격히 증가한 뒤 연결이 끊어지는 사건이 발생했습니다.

- 연결이 끊긴 시간: 03:06
- 현장 팀이 케이블 교체 및 커넥터 청소를 수행했습니다.
- 작업 완료 후 03:26에 연결이 정상 복구되었습니다.
- 복구 후 오류 횟수는 0건으로 확인되었습니다.

> 📌 **용어 설명**
> - **CRC 에러**: 데이터가 전송 중에 손상되었을 때 장비가 감지하는 오류 신호입니다.  
>   짧은 시간에 많이 발생하면 물리적 연결(케이블·커넥터) 문제를 의심합니다.
> - **Link Down**: 두 장비 사이의 연결이 끊어진 상태를 의미합니다.
> - **Gi0/1**: 스위치 장비에 있는 포트(입출력 구멍) 중 하나의 이름입니다.

## 6. 로그에서 확인된 사실

아래 내용은 샘플 로그 파일에서 직접 확인된 사실입니다.

1. 사건 발생 전(03:02) `SAMPLE_TOR_SW_01`은 정상 heartbeat 상태였습니다.
2. 03:05에 Gi0/1 포트의 CRC 에러 카운터가 5분 이내 **154회**로 증가했습니다.
3. CRC 에러 증가 **1분 후(03:06)** 같은 포트가 DOWN 상태가 되었습니다.
4. Ticket `EDU-TKT-2026-0003`이 자동으로 개설되었습니다.
5. 티켓이 현장 케이블링팀으로 이관되었습니다.
6. 현장 팀이 패치 코드 교체와 커넥터 청소를 수행했습니다.
7. 작업 완료 후 포트가 UP 상태로 복구되었습니다.
8. 복구 후 CRC 에러는 **0건**으로 확인되었습니다.

## 7. 가능한 원인

로그에서 확인된 사실을 바탕으로, 아래 세 가지 원인을 **가능성**으로 검토합니다.  
현재 로그만으로는 하나의 원인으로 확정할 수 없습니다.

### 🔹 가능성 A — 패치 케이블 또는 커넥터 접촉 불량

- **근거**: 조치 내용이 "패치 코드 교체"와 "커넥터 청소"였으며, 조치 후 문제가 해결되었습니다.
- **한계**: 교체 전 케이블·커넥터의 실제 상태를 로그만으로는 확인할 수 없습니다.

### 🔹 가능성 B — 케이블 자체의 물리적 손상

- **근거**: CRC 에러(데이터 손상 감지)가 Link Down보다 먼저 발생했습니다.  
  이는 연결이 끊기기 전부터 데이터 전송에 문제가 있었음을 나타냅니다.
- **한계**: 교체된 케이블이 실제로 손상되어 있었는지에 대한 점검 기록이 로그에 없습니다.

### 🔹 가능성 C — 포트(인터페이스) 자체의 하드웨어 문제

- **근거**: 로그만으로는 케이블 문제인지, 장비 포트 자체의 문제인지 구분할 수 없습니다.
- **한계**: 케이블만 교체하고 포트를 교체하지 않은 상태에서 문제가 해결되었으므로,  
  포트 문제일 가능성은 상대적으로 낮을 수 있으나 확정할 수 없습니다.

아래 항목은 샘플 로그에 기록되어 있지 않아 현재 알 수 없는 내용입니다.

- 교체 전 케이블이 실제로 손상되어 있었는지 여부

- 포트 자체의 하드웨어 결함 가능성 (케이블 교체만으로 해결됐으나 정확한 원인파악 불가)

## 9. 추가로 필요한 정보

향후 원인 분석을 더 정확하게 하려면 아래 정보가 필요합니다.

1. **같은 포트(Gi0/1)의 과거 이력**: 이전에도 CRC 에러나 Link Down이 발생한 적이 있는지
2. **케이블 점검 기록**: 교체 전 케이블의 외관 상태(육안 점검 결과 등)
3. **커넥터 상태 기록**: 청소 전 커넥터의 오염 또는 손상 여부
4. **인근 장비 동향**: 같은 시간대 주변 다른 장비에서도 유사한 이벤트가 있었는지
5. **작업자 추가 소견**: 현장 케이블링팀이 남긴 별도 작업 메모나 소견
