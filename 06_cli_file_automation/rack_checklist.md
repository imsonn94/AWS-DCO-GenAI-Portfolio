# [교육용 샘플 데이터 사용]

## 1. 개요 (Overview)
본 체크리스트는 교육용 샘플 티켓 및 로그 분석 시 점검하고 기록해야 할 항목을 정의한 서식입니다. 실제 장비 점검 절차가 아닌, 데이터 확인 및 기록 프로세스 훈련을 위해 사용됩니다.

## 2. 랙 확인 체크리스트 항목 (Checklist Items)

### [ ] 항목 1: 발생 시간 확인 (Verify Event Time)
- **가이드**: 수신된 샘플 티켓(`sample_ticket.md`) 또는 로그 분석 결과에 기록된 정확한 발생 시간을 확인하고 아래에 기록합니다.
- **기록란**: `2026-07-07 15:10:00 KST (SAMPLE)`

### [ ] 항목 2: 샘플 장비명 확인 (Verify Device Name)
- **가이드**: 이벤트가 발생한 장비의 명칭이 `SAMPLE_` 또는 `EDU_` 접두어를 사용하는 교육용 샘플 장비명인지 확인합니다.
- **기록란**: `SAMPLE_TOR_SW_01`

### [ ] 항목 3: 반복 발생 여부 확인 (Check for Recurring Issues)
- **가이드**: 동일한 샘플 장비 혹은 포트에서 유사한 CRC 에러나 Link Down 이벤트가 이전에도 발생했는지 과거 기록(SAMPLE_HISTORY)을 조회하고 기록합니다.
- **기록란**: [ ] 최초 발생 / [ ] 과거 반복 발생 이력 있음 (이전 티켓 ID: `EDU-TICKET-SAMPLE-XXXX`)

### [ ] 항목 4: Ticket 상태 확인 (Verify Ticket Status)
- **가이드**: 현재 해당 이슈에 대한 티켓 상태를 확인하고 올바르게 업데이트합니다.
- **기록란**: [ ] Open / [ ] In Progress / [ ] Resolved / [ ] Escalated

### [ ] 항목 5: 원인 단정 금지 (Do Not Assume Root Cause)
- **가이드**: 장애의 원인을 객관적인 근거 없이 성급하게 단정하지 마십시오. (예: "케이블이 고장났다" 대신 "CRC 에러 증가 후 Link Down이 발생하여 물리 계층 점검이 필요함"으로 사실 관계만 기술)
- **기록 확인**: [ ] 완수함 (객관적 사실 위주로 작성 완료)
