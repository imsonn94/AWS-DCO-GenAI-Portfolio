## 2. 내가 만든 DCO 인프라 용어 정의

### Rack

수많은 컴퓨터와 네트워크 장비들을 안전하고 깔끔하게 층층이 쌓아 올릴 수 있도록 만든 '거대한 규격화된 철제 선반' 또는 '책장'입니다.

### Server

24시간 내내 전 세계 사용자들에게 데이터나 서비스를 제공하기 위해 고안된 '고성능 컴퓨터'입니다. 일반 가정용 PC와 달리 모니터나 마우스 없이 오직 연산과 데이터 처리에만 집중하도록 얇고 납작한 상자 형태로 생겼습니다.

### NIC

컴퓨터가 인터넷(네트워크)망에 연결되어 데이터를 주고받을 수 있게 해주는 '통신 카드' 또는 '랜선 꽂는 구멍(포트)이 달린 부품'입니다.

### ToR Switch

하나의 선반(Rack) 맨 위쪽(Top)에 주로 설치되어, 그 선반에 있는 수십 대의 서버들을 하나로 묶어 외부 인터넷 세상과 연결해 주는 '대형 멀티탭 모양의 네트워크 중계 장치'입니다.

### Link Down

장비와 장비 사이를 연결하는 통신선(케이블)의 연결이 끊어져서 '데이터가 전혀 흐르지 않는 단선 상태'를 말합니다. 쉽게 말해 랜선이 뽑혔거나 선이 단절된 상황입니다.

### CRC error

데이터를 케이블을 통해 주고받는 과정에서, 선이 노후화되거나 찌그러져서 '데이터 중 일부가 깨지거나 오염되어 도착한 오류 상태'입니다. 말 전달 게임을 할 때 중간에 소음 때문에 말이 잘못 전달된 것과 비슷합니다.

### SSD failure

데이터를 저장하는 부품인 'SSD(고속 저장 장치)가 수명을 다하거나 고장 나서 읽기/쓰기가 불가능해진 현상'입니다.

### Fan Alert

서버나 장비 내부의 열을 식혀주는 '냉각 팬(선풍기 날개)이 멈추거나 회전 속도가 이상할 때 발생하는 경고 신호'입니다.

## 3. 헷갈리는 용어 비교

내가 비교한 용어: CRC error / SSD Failure

내가 이해한 차이: CRC error는 데이터를 실어 나르는 '통로(네트워크 케이블, 연결 포트 등)'의 품질 문제이며, SSD Failure는 데이터를 보관하는 '창고(저장 장치 하드웨어 부품)'의 수명 및 파손 문제입니다.

아직 헷갈리는 부분: 없음

## 4. 앞으로 사용할 학습 보조 프롬프트
프롬프트 1. 모르는 DCO 용어 설명용

•	역할: 데이터센터 운영(DCO) 직무를 처음 배우는 비전공자 대학생을 돕는 친절한 학습 도우미입니다.
•	목표: 아래 제시하는 DCO 기술 용어를 비전공자도 상식선에서 완벽히 직관적으로 이해할 수 있도록 쉽게 설명해 주세요.

•	상황: 저는 DCO 인턴십 직무 이해를 위한 생성형 AI 활용 수업을 듣고 있습니다. 실제 회사 내부 정보가 아닌, 공개된 직무 맥락과 교육용 예시로만 설명이 필요합니다.

•	출력 형식:
	1.	용어의 가장 쉬운 비유 (일상적인 사물이나 상황에 빗대어 1문장으로)
	2.	상세한 의미 설명 (3문장 이내로 명쾌하게)
	3.	이 용어가 DCO 현장 티켓(작업 지시서)에서 실제 어떻게 쓰일 수 있는지 가상의 교육용 예시 (장비명은 SAMPLE-DECO-01 형태의 가상 명칭 사용)
	4.	학생이 이 용어를 자기 글로 요약할 때 참고하면 좋은 핵심 단어 키워드들

•	제한 조건:
	1.	특정 기업의 실제 내부 절차처럼 단정하여 설명하지 마세요.
	2.	실제 티켓 번호, 실제 장비명, 실제 장비 시리얼 번호, 고객 개인 정보를 절대 임의로 만들어내지 마세요.
	3.	모든 예시는 SAMPLE, DEMO, EDU 같은 교육용 예시 표현만 사용하고, IP 주소가 필요하면 문서용 표준 대역(192.0.2.x 등)만 사용하세요.
	4.	제가 그대로 복사하기보다 자기 언어로 필기할 수 있도록 유도해 주세요.

프롬프트 2. 비슷한 용어 비교용

•	역할: 데이터센터 현장 운영(DCO) 인프라 용어 중 학생들이 자주 헷갈려하는 개념들을 명확하게 구분해 주는 비교 분석 코치입니다.

•	목표: 아래 제시하는 두 개의 DCO 용어가 가진 명확한 물리적·기능적 차이점을 비전공자도 이해하기 쉽게 대조하여 설명해 주세요.

•	상황: 저는 두 용어가 기능이나 발생 위치 면에서 비슷하게 느껴져서 혼동을 겪고 있습니다. 공개된 기술 맥락을 기준으로 교육용 초안을 작성해 주세요.

•	출력 형식:
	1.	용어 A의 핵심 정의와 주로 발생하는 물리적 위치
	2.	용어 B의 핵심 정의와 주로 발생하는 물리적 위치
	3.	두 용어의 가장 결정적인 차이점 (표 또는 한 줄 요약)
	4.	상황별 교육용 가상 시나리오 (예시 기재 시 DEMO-SERVER-A 형태의 표현 사용)
	5.	비전공자 학생이 두 개념을 머릿속에 각인할 수 있는 암기 팁/연상 기억법
	6.	현장에서 이 두 문제를 진단할 때 원인을 성급하게 하나로 단정하면 안 되는 이유(주의할 점)

•	제한 조건:
	1.	실제 인프라 기업의 보안 규정이나 내부 가이드를 아는 것처럼 단정적인 어조로 쓰지 마세요.
	2.	가상의 IP나 일련번호 등을 진짜처럼 꾸며내지 말고 오직 교육용 샘플로만 처리해 주세요.
	3.	학생이 직접 퀴즈를 풀듯 생각할 수 있는 여지를 문장 마지막에 남겨주세요.

프롬프트 3. 장애 표현 확인 순서 정리용

•	역할: 데이터센터 하드웨어/네트워크 장애 알람을 분석하고, 현장 엔지니어가 취해야 할 논리적 점검 순서를 설계해 주는 DCO 트러블슈팅 매니저입니다.

•	목표: 제시된 가상의 장애 증상(현상)을 보고, 발생 가능한 물리적 원인들과 안전을 고려한 표준적인 확인 순서 가이드를 작성해 주세요.

•	입력: [공부하고자 하는 장애 알람 문구나 현상을 여기에 입력하세요]

•	출력 형식:
	1.	알람 현상 분석 (이 알람이 구체적으로 어떤 물리적 위험을 경고하는가)
	2.	예측 가능한 가상 원인 3가지 (소프트웨어적, 하드웨어적, 환경적 요인을 열어두고 작성)
	3.	현장 출동 시 엔지니어가 밟아야 할 논리적 확인 순서 (가장 단순하고 안전한 물리적 점검부터 시작하는 순서로 단계별 작성)
	4.	연계되는 DCO 필수 개념 (이 작업을 할 때 함께 알아야 할 SOP, Escalation 등과의 연관성)
	5.	[현장 안전 및 보안 경고]: 인턴십 지원자 수준에서 명심해야 할 주의사항
•	제한 조건:
	1.	특정 회사의 전용 툴 이름이나 실제 티켓 시스템의 스크린샷을 모사하는 정보를 넣지 마세요.
	2.	모든 상황은 EDU-SCENARIO-01과 같이 가상의 실습용 환경임을 명시해 주세요.
	3.	원인과 해결책을 하나로 단정 짓지 말고, 논리적인 가설 검증 프로세스 중심으로 작성해 주세요.
